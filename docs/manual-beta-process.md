# Manual Beta Process

## Objetivo

Validar demanda antes de gastar com backend complexo ou APIs caras.

## Fluxo beta manual

1. Usuário escolhe jogo.
2. Cabine Pro gera briefing com dados simulados/manualmente verificados.
3. João revisa fontes e riscos.
4. Usuário recebe link/PDF/Modo Cabine.
5. Usuário dá feedback via issue/template.
6. Aprendizados viram ajustes no produto.

## Métricas

- tempo economizado;
- confiança percebida;
- uso antes de jogo real;
- partes mais usadas;
- partes ignoradas;
- willingness to pay;
- erros encontrados;
- fontes mais necessárias.

## Critério para sair do beta manual

- 5–10 usuários testaram;
- 2+ dizem que pagariam;
- 1+ usa em preparação real;
- principais riscos de fonte estão mapeados;
- preço inicial validado;
- custo por briefing estimado.
