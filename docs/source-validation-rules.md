# Regras de Fontes e Confiança

## Princípio

A IA não é fonte da verdade. Ela só pode escrever como fato aquilo que está ancorado em evidência rastreável.

## Níveis de confiança

### 1. Confirmado

Pode ser usado ao vivo como fato.

Exemplos:

- site oficial do clube;
- federação/confederação;
- competição;
- súmula;
- regulamento;
- coletiva oficial;
- API licenciada com permissão de uso;
- documento oficial.

### 2. Alta confiança

Pode ser usado com contexto, desde que não exista conflito.

Exemplos:

- provedor de dados confiável;
- veículo editorial forte;
- duas ou mais fontes independentes;
- setorista reconhecido + evidência adicional.

### 3. Verificar

Não deve ser cravado ao vivo sem checagem final.

Exemplos:

- provável escalação;
- informação de treino;
- lesão não confirmada oficialmente;
- bastidor;
- matéria de jornal local;
- fonte única.

### 4. Cuidado

Deve ficar fora do Modo Seguro.

Exemplos:

- rumor;
- rede social isolada;
- conflito entre fontes;
- informação sem data;
- fonte sem reputação;
- conteúdo possivelmente copiado ou protegido.

## Score do briefing

- **95–100%:** pronto para transmissão;
- **85–94%:** usável com revisão;
- **70–84%:** rascunho;
- **abaixo de 70%:** não usar ao vivo.

## Regras de bloqueio

O briefing não pode aparecer como “pronto para transmissão” se existir:

- conflito crítico não resolvido;
- escalação tratada como confirmada sem fonte oficial;
- lesão tratada como confirmada sem fonte oficial;
- dado de tabela/jogo sem fonte;
- fonte proibida ou scraping não autorizado;
- uso de conteúdo protegido copiado.

## Modo Seguro

O Modo Seguro só exibe:

- Confirmado;
- Alta confiança sem conflito;
- dados com horário de atualização;
- textos sem rumor ou bastidor frágil.

Tudo que for “Verificar” ou “Cuidado” deve ficar oculto, sinalizado ou removível.
