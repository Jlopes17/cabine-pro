# Segurança e Privacidade

## Status do protótipo

O protótipo atual é local/estático. Nenhum dado é enviado para servidor.

## Produto real

Antes de vender oficialmente, o produto deve ter:

- autenticação segura;
- senha armazenada via provedor confiável, nunca em texto puro;
- opção de login social ou SSO no futuro;
- controle de acesso por equipe;
- permissões por papel: owner, admin, editor, viewer;
- logs de auditoria;
- política de retenção de dados;
- exclusão de conta/dados;
- criptografia em trânsito;
- backup;
- monitoramento de erro;
- proteção contra prompt injection;
- isolamento de fontes não confiáveis;
- revisão LGPD/GDPR.

## Dados pessoais prováveis

- nome;
- email;
- dados de cobrança;
- histórico de briefings;
- preferências de times/competições;
- logs de uso;
- membros de equipe.

## Segurança editorial

Segurança no Cabine Pro não é só login. Também significa não colocar o usuário em risco ao vivo.

O produto deve:

- mostrar fonte por informação crítica;
- sinalizar conflitos;
- ocultar rumores no Modo Seguro;
- registrar atualização e revisão;
- nunca afirmar como confirmado sem evidência.

## Pendências antes de lançamento comercial

- revisar política de privacidade com advogado;
- definir controlador/operador de dados;
- definir DPA com fornecedores;
- definir retenção e exclusão de dados;
- definir termos para uso de IA e prompts;
- definir política para dados de terceiros.
