# GitHub Validation Plan

## Objetivo

Usar o GitHub como centro de validação do produto antes de transformar a Cabine Pro em SaaS pago.

## Milestones sugeridos

### Milestone 1 — Prototype QA

Validar que o protótipo completo funciona em desktop e mobile.

Issues:

- Validar landing page;
- Validar login demo;
- Validar criação de briefing;
- Validar briefing completo;
- Validar modo cabine;
- Validar fontes;
- Validar auditoria;
- Validar planos e políticas.

### Milestone 2 — Validação com usuário real

Testar com narradores/produtores.

Perguntas:

- Usaria antes de entrar ao vivo?
- Quanto tempo economizou?
- Quais blocos são indispensáveis?
- O que falta para confiar?
- Pagaria? Quanto?

### Milestone 3 — Trust Model

Validar score, Modo Seguro e fontes.

Entregáveis:

- fórmula de score;
- matriz de fontes;
- regra de bloqueio;
- auditoria;
- logs de revisão;
- checklist antes do ar.

### Milestone 4 — MVP técnico

Definir stack e arquitetura.

- Next.js;
- Supabase/Postgres;
- Auth;
- OpenAI;
- API esportiva licenciada;
- pagamento;
- emails;
- monitoramento.

### Milestone 5 — Beta pago

Só iniciar depois de validação real.

Critérios:

- pelo menos 5 usuários reais testaram;
- pelo menos 2 dizem que pagariam;
- pelo menos 1 briefing foi usado em preparação real;
- riscos de fonte mapeados;
- custo mensal controlado.

## Regra de ouro

Não construir backend complexo antes de provar que o briefing é útil e confiável.
