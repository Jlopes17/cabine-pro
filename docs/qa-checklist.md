# QA Checklist — Cabine Pro

## Landing page

- [ ] Valor entendido em menos de 10 segundos.
- [ ] Foco em confiança claro.
- [ ] CTA principal visível.
- [ ] Disclaimer editorial acessível.
- [ ] Planos/preços não prometem mais do que o produto entrega.

## Login e onboarding

- [ ] Login demo funciona.
- [ ] Termos/disclaimer aparecem antes do uso.
- [ ] Perfil do usuário é claro.
- [ ] Fluxo não trava no mobile.

## Briefing

- [ ] Resumo de 1 minuto existe.
- [ ] Ficha da partida existe.
- [ ] Situação da competição existe.
- [ ] Times e personagens existem.
- [ ] Histórico existe.
- [ ] Mercado/contexto existe.
- [ ] Narrativas existem.
- [ ] Escalações e alertas existem.
- [ ] Fontes aparecem.
- [ ] Exportação PDF é possível.

## Confiança

- [ ] Score mostra explicação.
- [ ] Status “Pronto para transmissão” não aparece sem critério.
- [ ] Informações críticas têm fonte.
- [ ] Rumores não entram como fatos.
- [ ] Conflitos aparecem.
- [ ] Alertas críticos são visíveis.

## Modo Cabine

- [ ] Texto legível.
- [ ] Aumentar/diminuir fonte funciona.
- [ ] Copiar abertura funciona.
- [ ] Modo Seguro remove itens frágeis.
- [ ] Checklist antes do ar é útil.

## Mobile

- [ ] iPhone Safari.
- [ ] Android Chrome.
- [ ] Botões grandes o suficiente.
- [ ] Navegação inferior funciona.
- [ ] Não existe conteúdo escondido.

## Comercial

- [ ] Planos são compreensíveis.
- [ ] Cancelamento está descrito.
- [ ] Limites de uso estão claros.
- [ ] O usuário entende que é apoio editorial.

## Validação com narrador/produtor

- [ ] Ele usaria antes de um jogo real?
- [ ] Quanto tempo economizou?
- [ ] O que ele ainda faria manualmente?
- [ ] Qual parte gerou mais confiança?
- [ ] Qual parte gerou dúvida?
- [ ] Pagaria? Quanto?
